Ruby Classes

Keeping it Classy
2015-04-19

Points of this Blog

- Explain when, why, and how to use classes
- Explain how data structures and classes can model a real-world object

Classes are like blue prints for new objects you want to create. If I wanted to make a host of objects with similar properties, creating a Class with the desired properties is the way to go. Creating a Class in Ruby is simple, the syntax is: 


          def MyClass
          end

A Class will usually have methods that lets you interact with objects created by the class. 


          def MyClass

            def greeting
              puts "Hello!"
            end

          end

In the example above, any object created from the "MyClass" class will have a greeting method that will print out "Hello!" when called.

Classes can be used to model real-world objects, I have created a Car class below to model cars. 


        def Cars

          def initialize(aName)
            @name = aName
          end

          def start
            puts "The car grumbled to life."
          end

          def move
            puts "The car roars forward."
          end

          def stop
            puts "The car screeches to a halt"
          end

        end

To create a car using the above class, simple run: 


        car = Cars.new("Marge")

As you can see, the car created from the Car class above can start, move, and stop. The car will each also have the name "Marge" (because what car doesn't have a name?).

Now that we have a basic blue print of a car, how will we model different types of cars? Sure, all cars start, move, and stop (at least I hope they can), but what about electric cars or gasoline cars? This is where subclasses come in. Instead of creating more classes called "ElectricCars" and "GasolineCars" with the same start, move, stop methods, we can simply make subclasses. 


        def GasolineCars < Cars

          def power_source
            puts "Gas"
          end

        end

        def ElectricCars < Cars

          def power_source
            puts "Electric"
          end

        end

The "ElectricCars" and "GasolineCars" classes will have the same start, move, stop methods as the "Cars" method. The "ElectricCars" and "GasolineCars" will also have a new method "power_source". Although you can create as many methods as you want within a single class, it is good practice to keep each class serving a specific function. You can always create subclasses to add specific functionalities.
